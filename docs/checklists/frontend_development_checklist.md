# 前端开发检查清单

本文档提供了前端开发的完整检查清单，确保功能开发一次性成功。

## 开发前准备

- [ ] 需求分析完成
  - [ ] 页面功能定义明确
  - [ ] API 接口确认（后端接口已开发完成）
  - [ ] UI/UX 设计确认

- [ ] 开发环境检查
  - [ ] 本地服务运行正常
  - [ ] API 接口可访问

## 前端开发阶段

- [ ] 前端页面开发完成
  - [ ] 文件位置：`local_frontend/xxx.html`
  - [ ] HTML 结构正确
  - [ ] CSS 样式正确
  - [ ] 响应式设计（如需要）

- [ ] API 调用正确
  - [ ] 使用 gRPC-Web 调用 API（通过 `api.js`）
  - [ ] 使用 `api.post('/xxx/endpoint', data)` 调用
  - [ ] 错误处理正确（显示错误信息）
  - [ ] 加载状态显示正确

- [ ] 错误处理必须显示 UI 区域
  - [ ] 错误处理函数同时更新内容和显示状态
  - [ ] 如果 UI 区域初始为 `display:none`，错误处理时必须显示
  - [ ] 错误处理逻辑与正常流程一致

- [ ] 关键阶段提前显示 UI 区域
  - [ ] 在进入关键处理阶段时，提前显示相关 UI 区域
  - [ ] 确保用户能看到处理进度和结果

- [ ] 展示逻辑完成
  - [ ] 文件位置：`local_frontend/js/xxx.js`
  - [ ] 数据渲染正确
  - [ ] 交互逻辑正确
  - [ ] 事件处理正确

- [ ] 代码规范检查通过
  - [ ] 无硬编码路径
  - [ ] 无 XSS 风险（使用 textContent 或 DOMPurify）
  - [ ] 错误处理完善

## 测试阶段

- [ ] 前端功能测试通过（手动）
  - [ ] 页面可以正常访问
  - [ ] API 调用正常
  - [ ] 数据展示正确
  - [ ] 错误处理正确
  - [ ] 交互逻辑正确

- [ ] 浏览器兼容性测试（如需要）
  - [ ] Chrome 测试通过
  - [ ] Firefox 测试通过
  - [ ] Safari 测试通过（如需要）

- [ ] 响应式测试（如需要）
  - [ ] 移动端显示正常
  - [ ] 平板端显示正常
  - [ ] 桌面端显示正常

## 代码审查阶段

- [ ] 代码审查通过
  - [ ] 无硬编码路径
  - [ ] 无 XSS 风险
  - [ ] 错误处理完善
  - [ ] 代码规范符合要求

## 部署阶段

- [ ] 代码已提交到 Git
  ```bash
  git add .
  git commit -m "feat: 新增前端页面 xxx"
  ```

- [ ] 代码已推送到 GitHub
  ```bash
  git push origin master
  ```

- [ ] 增量部署执行
  ```bash
  python3 scripts/deploy/auto_deploy.py --mode incremental
  ```

- [ ] 部署验证通过
  - [ ] 页面可以正常访问
  - [ ] API 调用正常
  - [ ] 数据展示正确
  - [ ] 错误处理正确

## 完成标准

所有检查项必须全部通过，才能认为功能开发完成。

**成功标准**：
- ✅ 所有检查项通过
- ✅ 前端功能测试通过
- ✅ 部署验证通过
- ✅ 页面在生产环境正常运行
