# 文档归档目录

本目录用于归档已解决的历史问题、测试报告、代码审查报告等文档。

## 目录结构

- `resolved_issues/` - 已解决的问题文档和修复报告
- `test_reports/` - 历史测试报告
- `code_reviews/` - 代码审查报告
- `optimization_reports/` - 优化报告

## 归档原则

1. **已解决的问题**：问题已解决且不再需要参考的文档
2. **历史测试报告**：过时的测试报告，保留作为历史记录
3. **代码审查报告**：已完成审查的代码审查报告
4. **优化报告**：已完成的优化工作记录

## 维护说明

- 归档文档按时间或主题分类
- 重要的问题复盘应保留在 `docs/knowledge_base/problem_history.md` 中作为摘要
- 归档文档不再更新，仅作为历史记录
