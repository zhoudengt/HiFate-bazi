# 旺衰从5级改为3级 - 修改说明

## 修改日期
2026-03-02

## 修改概述
将命局旺衰从 5 级（极旺、身旺、平衡、身弱、极弱）合并为 3 级（身旺、平衡、身弱），并同步更新喜忌神配置及所有依赖逻辑。

## 变更映射

| 原 5 级 | 新 3 级 | 喜忌（xi_ji） |
|---------|---------|---------------|
| 极旺 | 身旺 | 喜：食神,伤官,偏财,正财,七杀,正官；忌：比肩,劫财,偏印,正印 |
| 身旺 | 身旺 | 同上 |
| 平衡 | 平衡 | 喜：比肩,劫财,食神,伤官；忌：偏财,正财,七杀,正官 |
| 身弱 | 身弱 | 喜：比肩,劫财,偏印,正印；忌：食神,伤官,偏财,正财,七杀,正官 |
| 极弱 | 身弱 | 同上 |

## 修改的文件

### 1. core/data/wangshuai_config.py
- **thresholds**：从 5 条改为 3 条
  - 身旺：60-100 分
  - 平衡：40-59 分
  - 身弱：0-39 分
- **xi_ji**：从 5 条改为 3 条，喜忌神配置与上表一致

### 2. core/analyzers/wangshuai_analyzer.py
- **_determine_wangshuai()**：判定逻辑从 5 档合并为 3 档
  - 总分 > 10 → 身旺（原极旺+身旺）
  - -10 ≤ 总分 ≤ 10 → 平衡
  - 总分 < -10 → 身弱（原身弱+极弱）
  
- **_calculate_wangshuai_degree()**：量化评分映射调整
  - 身旺：60-100 分（总分越高，分数越高）
  - 平衡：40-60 分（以0分为中心）
  - 身弱：0-40 分（总分越低，分数越低）

### 3. server/services/fortune_scoring_service.py
- 移除了对"极旺"/"极弱"的特殊处理
- 改为对"身旺"/"身弱"做温和的扣分（-0.3分）
- 对"平衡"状态加分（+0.3分）

### 4. server/api/v1/wangshuai.py
- 更新 API 文档字符串，说明最终判定为 3 级制：身旺、平衡、身弱

## 兼容性说明

### 向后兼容
- 接口契约：`wangshuai` 字段由 5 种枚举变为 3 种
- 原返回"极旺"的用户现在会得到"身旺"
- 原返回"极弱"的用户现在会得到"身弱"
- 喜忌神配置保持一致（极旺与身旺相同，极弱与身弱相同）

### 缓存处理
旺衰结果有 Redis 缓存（约 30 天），修改后需要：
- **方案 A**：部署时清空 `wangshuai` 相关 key
- **方案 B**：为缓存 key 增加版本号（如 `wangshuai:v2`）

**建议**：在部署时执行以下命令清理旧缓存：
```bash
redis-cli KEYS "wangshuai:*" | xargs redis-cli DEL
```

## 不受影响的部分

1. **core/analyzers/wuxing_balance_analyzer.py**：其中的"极旺"/"极弱"用于**五行强弱**（金木水火土），与**命局旺衰**不同，保持不变

2. **训练数据生成器**（tools/training_data/）：示例数据使用的是"身旺"、"身弱"，在 3 级制中仍然有效

3. **规则条件**（standards/rule-development.md）：`wangshuai` 条件已支持 `["身旺"]`/`["身弱"]`/`["平衡"]`，无需修改

4. **前端**（local_frontend/）：由前端团队维护，后端接口返回的 `wangshuai` 枚举值变更后，前端若依赖旧值需前端配合

## 部署建议

1. 提交代码前先在本地测试
2. 部署到 Node2（测试环境）验证
3. 清理 Redis 旧缓存
4. 运行回归测试确保所有接口正常
5. 部署到 Node1（生产环境）
